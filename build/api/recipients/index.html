Recipient API

Object Attributes

Name Type Description
object string

The string recipient.
id string

The RECIPIENT_ID.
livemode string

Whether this is a live (true) or test (false) recipient.
location string

Path to retrieve the recipient.
verified boolean

Whether the recipient is verified or not
active boolean

Whether the recipient is active or not
name string

The recipient's name.
email string

The recipient's email.
description string

The recipient description.
type string

The recipient type individual or corporation
tax_id string

The recipient tax id
bank_account bank_account

The recipient's bank account
failure_code string

If the recipient could not be verified, a failure code explaining why.
created datetime

Creation date of the recipient in ISO 8601 format.

Example

  • Default JSON Representation 

    {
  "object": "recipient",
  "id": "recp_test_5086xmr74vxs0ajpo78",

  "verified": false,
  "active": false,
  "name": "James Smith",
  "email": "secondary@recipient.co",
  "description": "Secondary recipient",
  "type": "individual",
  "tax_id": "1234567890",

    "object": "bank_account",

List all recipients

- GET https://api.omise.co/recipients

Query Parameters

Name Type Description
offset integer

(optional, default: 0) The offset of the first record returned. I.e.: How many records to skip from the beginning.
limit integer

(optional, default: 20, maximum: 100) The maximum amount of records returned.
from datetime

(optional, default: 1970-01-01T00:00:00Z, format: ISO 8601) The UTC date and time limiting the beginning of returned records. E.g.: 2014-10-20T00:00:00Z
to datetime

(optional, default: current UTC Datetime, format: ISO 8601) The UTC date and time limiting the end of returned records. E.g.: 2015-01-20T00:00:00Z
